FAQ

A pronounceable password uses easy-to-pronounce syllables to form memorable strings. When enabled, other options like symbols or patterns are disabled to keep it simple.
Use letters to define structure:
  • L: Letter (A-Z or a-z)
  • N: Number (0–9)
  • S: Symbol (e.g., @, #, $)
  • !: Any character
  • Other characters are treated literally (e.g., dashes or spaces)
Example: L-N-S-L! generates a letter, dash, number, symbol, letter, and any character.
A passphrase is a set of easy-to-remember words separated by dashes (like: sunny-dragon-planet-tiger). It’s secure if it’s long enough.
The “Include” field ensures those characters appear in the password. The “Exclude” field removes those characters from the character pool. Note: If the same character is in both, it will be excluded.
  • No Repeat: Prevents characters from being reused in the password.
  • No Sequential: Blocks common sequences like abc or 123.
Characters like l, 1, I, 0, and O can look similar and cause confusion. Excluding them makes your password easier to read and type correctly.
